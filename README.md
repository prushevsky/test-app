# Web Testing System

Простое веб-приложение для создания и прохождения тестов с административной панелью.

## Быстрый запуск

### 1. Установка зависимостей
```bash
pip install flask python-dotenv
```

### 2. Создание учетных данных администратора

Запустите генератор пароля и следуйте инструкциям:
```bash
python generate_password_hash.py
```

Или создайте файл `.env` вручную:
```
ADMIN_USERNAME=admin
ADMIN_PASSWORD_HASH=ваш_хэш_пароля_здесь
```

Для тестирования можете использовать стандартный пароль `admin123` (меняйте в продакшене!).

### 3. Запуск приложения
```bash
python app.py
```

Откройте в браузере: `http://localhost:5000`

## Основные возможности

### Для пользователей:
- Пройти тест, введя свое имя
- Посмотреть результаты сразу после завершения
- Просмотреть историю всех своих тестов

### Для администраторов (логин: admin):
- Просматривать и управлять вопросами (добавлять/удалять)
- Смотреть статистику ответов
- Просматривать логи действий пользователей
- Менять пароль администратора

## Структура файлов

```
app.py                    # Основной файл Flask
generate_password_hash.py # Генератор хэша пароля
.env                      # Файл с учетными данными (создать)
templates/                # HTML страницы
static/script.js          # JavaScript для клиентской части
data/                     # Все данные хранятся здесь
  ├── questions.json      # Вопросы теста
  ├── results.json        # Результаты тестирования
  └── user_actions.json   # Логи действий
```

## Как работает

1. **При запуске** создаются необходимые папки и файлы, если их нет
2. **Пользователь** заходит на главную страницу, вводит имя, отвечает на вопросы
3. **Результаты** автоматически сохраняются и показываются
4. **Администратор** может войти по паролю и управлять системой
5. **Все действия** логируются в файлы для отладки

## Безопасность

- Пароли хэшируются с солью
- Ограничение попыток входа (3 попытки, блокировка на 5 минут)
- Таймаут сессии администратора (30 минут)
- Все действия записываются в логи с IP адресами

## Отладка

- **Ctrl + D** на странице теста - показать отладочную информацию
- **Ctrl + Shift + R** в админке - перезагрузить данные
- Логи доступны в `/admin/logs` после входа администратора

## Важные файлы

- `data/questions.json` - редактируйте здесь вопросы напрямую
- `data/results.json` - здесь хранятся все результаты
- `data/user_actions.json` - все действия пользователей

Приложение готово к работе сразу после установки зависимостей. Все данные хранятся в папке `data/` в формате JSON.
